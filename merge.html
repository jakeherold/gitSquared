Git Merge: Brings Everyone Up-to-date

git merge [target branch]: merge the target branch into the current branch (see which branch you are on by calling git branch). Git will determine the merge algorithm automatically (discussed below).

git merge --no-ff [branch]: merge the target branch into the current branch, always generate a merge commit (even if it was a fast-forward merge). This is useful for documenting all merges that occur in your repository.

Note:

  • All commands listed above merge the target branch into the current branch that you are working on.
  • The current branch will be updated.
  • The target branch is completely unaffected.
  • Merge commands are often used with git checkout and git branch -d
  • There are two types of merge: fast-forward merge and 3-way merge.

Fast Forward Merge:

  • Occurs when there is no change applied to the parent branch eversince you created a new child branch from it and added changes to that child branch
  • Git combines the histories of the two branches, since all of the commits reachable from the target branch are now available through the current one.

3-Way Merge:

  • You have to go through a 3-way merge when the branches have diverged.
  • 3-way merge uses three commits to generate the merge commit: one from the parent's tip, one from the child's tip and the third one is from the point right before the child started to grow.
  • This resembles tying the two branches back together.

Conflicts-The root of all the pains

  • This happens when the two branches have changes at the same part of a file.
  • Git does not know which change should be kept and which change should be discarded.
  • When this occurs, it stops right before the merge commit and you need to resolve the conflicts manually.
  • Running git status will tell you which file(s) resulted in conflicts
  • After resolving the conflicts and ready to continue merging, calling git add will tell Git that the conflicts have been resolved.
  • Run git commit will then generate the merge commit.
